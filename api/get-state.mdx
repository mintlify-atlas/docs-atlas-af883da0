## Overview

The `get_state` method returns a snapshot of the current application state, including the active headline, applied filters, and pool statistics.

## Method Signature

```javascript
await window.Neckass.agent.call('get_state')
```

## Parameters

This method takes no parameters.

## Return Value

Returns a state snapshot object:

<ResponseField name="index" type="number | null">
  The current headline's position in the internal headlines array. Returns `null` if no valid headline is loaded.
</ResponseField>

<ResponseField name="identifier" type="string | null">
  A stable identifier string for the current headline:
  - `"curated-{N}"` for curated headlines from the catalog
  - `"generated-{N}"` for headlines created by the tiny model
  - `null` if no headline is active
</ResponseField>

<ResponseField name="headline" type="string">
  The current headline text. Returns an empty string if no headline is loaded.
</ResponseField>

<ResponseField name="url" type="string | null">
  The canonical URL for the current headline, including query parameters for filters and the headline identifier. Returns `null` if no headline is active.
</ResponseField>

<ResponseField name="filters" type="object">
  The currently active filter settings:
  
  <Expandable title="Filter properties">
    <ResponseField name="section" type="string">
      Active section filter (e.g., "latest", "local", "politics", "sports")
    </ResponseField>
    <ResponseField name="query" type="string">
      Active search query string
    </ResponseField>
    <ResponseField name="source" type="string">
      Source preference: "auto" (mix), "curated" (catalog only), or "generated" (tiny model only)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="poolSize" type="number">
  The number of headlines that match the current filters. This is the size of the eligible headline pool for shuffle operations.
</ResponseField>

<ResponseField name="totalHeadlines" type="number">
  The total number of headlines available, including both curated headlines and any headlines generated by the tiny model during this session.
</ResponseField>

<ResponseField name="generatorAvailable" type="boolean">
  Whether the tiny model headline generator is available and functional in the current environment.
</ResponseField>

## Usage Examples

### Basic State Retrieval

```javascript
const state = await window.Neckass.agent.call('get_state');

console.log(`Current headline: ${state.headline}`);
console.log(`Pool size: ${state.poolSize}`);
console.log(`Total headlines: ${state.totalHeadlines}`);
```

### Checking Active Filters

```javascript
const state = await window.Neckass.agent.call('get_state');

if (state.filters.section !== 'latest') {
  console.log(`Viewing ${state.filters.section} headlines`);
}

if (state.filters.query) {
  console.log(`Searching for: ${state.filters.query}`);
}
```

### Monitoring Generator Availability

```javascript
const state = await window.Neckass.agent.call('get_state');

if (!state.generatorAvailable && state.filters.source === 'generated') {
  console.warn('Generator filter is active but generator is unavailable');
}
```

### Validating State Before Actions

```javascript
const state = await window.Neckass.agent.call('get_state');

if (state.index === null) {
  console.log('No headline loaded, shuffling...');
  await window.Neckass.agent.call('shuffle');
} else {
  console.log(`Headline ${state.identifier} is loaded`);
}
```

### Tracking Pool Size Changes

```javascript
const before = await window.Neckass.agent.call('get_state');
console.log(`Pool size before filter: ${before.poolSize}`);

await window.Neckass.agent.call('set_filters', { section: 'local' });

const after = await window.Neckass.agent.call('get_state');
console.log(`Pool size after filter: ${after.poolSize}`);
```

## Implementation Details

The method calls `app.getAgentSnapshot()` internally:

```javascript
getAgentSnapshot() {
    const currentIndex = isValidHeadlineIndex(this.state.currentIndex, this.headlines.length)
        ? this.state.currentIndex
        : null;
    return {
        index: currentIndex,
        identifier: currentIndex === null ? null : this.identifierFromIndex(currentIndex),
        headline: this.getCurrentHeadlineText(),
        url: currentIndex === null ? null : this.getCanonicalUrl(currentIndex),
        filters: { ...this.filters },
        poolSize: this.filteredIndexes.length,
        totalHeadlines: this.headlines.length,
        generatorAvailable: this.isGeneratorAvailable()
    };
}
```

Source: `modules/agent-interface.js:10-12`, `modules/app.js:1703-1717`

## State Persistence

The state snapshot reflects the current in-memory application state. The following data is persisted to `localStorage`:

- Navigation history stack
- Active filters
- Generated headlines
- Unique headlines viewed
- Favorites (if feature is enabled)

## Error Handling

This method does not throw errors under normal operation. It always returns a valid state snapshot, even when:

- No headlines are loaded
- Filters exclude all headlines
- The generator is unavailable

In edge cases, fields will be set to safe defaults (`null`, empty string, `0`).

## Performance Notes

- This is a synchronous snapshot operation with minimal overhead
- Filter object is cloned to prevent external mutation
- Safe to call frequently for state monitoring
