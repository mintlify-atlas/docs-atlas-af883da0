## Key Features

### Instant Headline Generation
Click **Shuffle** to cycle through featured headlines. Use **Previous** to navigate backward through your session history. Headlines are randomly selected from a curated catalog, with optional generated headlines from a tiny on-device LLM mock.

### Copy, Share, Export
- **Clipboard copy**: Copy the current headline with one click
- **Social share links**: Pre-filled share links for Twitter, Facebook, and Reddit
- **Native share**: Use your device's share sheet to share anywhere
- **Mock front page export**: Download or copy a styled PNG image of the headline as a satirical front page

### Session Tracking
Your app automatically tracks:
- **Headlines viewed**: Count of unique headlines you've seen
- **Recent headlines**: Navigate back through your session history
- **Daily streaks**: Encouragement to return daily
- **Shuffle streaks**: Track how many headlines you've shuffled in one session

### Filter & Customize
Filter headlines by section, search query, or source (curated vs. generated) using URL parameters. Clear filters with one click to reset to all headlines.

### Agentic Interface
A lightweight browser-side tool interface is exposed at `window.Neckass.agent` for browser automation and agentic LLM runtimes:

```js
const state = await window.Neckass.agent.call('get_state');
const next = await window.Neckass.agent.call('shuffle');
const previous = await window.Neckass.agent.call('previous');
```

Available tools: `get_state`, `shuffle`, `previous`, `generate`, `set_filters`, `clear_filters`, `select_headline`, `list_headlines`.

## No Build Required

Neckass Headlines is designed to work without any build tooling:

- No npm install required for runtime
- No webpack, vite, or bundler
- No transpilation or compilation step
- Just open the HTML file and go

Optional smoke tests use Playwright (`npm install` + `npm run test:smoke`) but are not required to run the app.
